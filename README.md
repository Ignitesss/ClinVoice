# ClinVoice - сервис распознавания речи для медицинских консультаций

Приложение для врачей: запись **одним потоком** с микрофона через **WebRTC**; аудио накапливается для **Whisper**. При настроенных ключах Yandex дополнительно показывается **черновик SpeechKit**; протокол строится **YandexGPT** только по тексту Whisper.

## Установка

```bash
pip install -r requirements.txt
streamlit run app.py
```

Либо просто перейдите по ссылке, чтобы попробовать наш сервис - https://clinvoice-d6qrtxccwn8gy5zuxvnbx5.streamlit.app/

## Переменные окружения (Yandex Cloud)

Для **протокола** и **черновика SpeechKit** используются одни и те же учётные данные, что и в `protocol.py`

## Возможности

- **Микрофон:** один блок WebRTC — включите запись, ведите консультацию, остановите при необходимости. Звук в 16 kHz mono копится в буфер для Whisper (лимит объёма задаётся `CLINVOICE_MAX_PCM_BYTES`, по умолчанию ~80 МБ).
- **Черновик SpeechKit** (если задан API-ключ или IAM): подсказка по мере речи, не участвует в протоколе.
- **Whisper:** по накопленному аудио после кнопки «Транскрибировать…»; длинные записи режутся на части (`CLINVOICE_ASR_CHUNK_SECONDS`, по умолчанию 30 с).
- **Скачивание:** **protocol.docx** и **protocol.txt**; после успешного заполнения протокола пробуется автозагрузка `protocol.txt` в браузере.

## Требования к системе

- Python 3.8+
- ~2GB disk space for Whisper models

## Как пользоваться?

1. Включите запись в блоке WebRTC, говорите; при необходимости остановите и снова включите (буфер Whisper дополняется).
2. Нажмите **«Транскрибировать и заполнить протокол»** (нужно хотя бы ~1 с речи в буфере).
3. Проверьте транскрипт и протокол, при необходимости отредактируйте.
4. Сохраните протокол в **.docx** или **.txt** (или дождитесь автозагрузки `.txt`).
5. Кнопка **«Сбросить запись и черновик»** очищает буфер и черновик без транскрибации.
