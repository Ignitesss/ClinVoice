# ClinVoice - сервис распознавания речи для медицинских консультаций

Приложение для врачей: **черновик** во время диктовки через **Yandex SpeechKit** (поток из браузера по WebRTC) и **итоговый транскрипт** через **Whisper** по записанным WAV-фрагментам; протокол заполняется **YandexGPT** только по тексту Whisper.

## Установка

```bash
pip install -r requirements.txt
streamlit run app.py
```

Либо просто перейдите по ссылке, чтобы попробовать наш сервис - https://clinvoice-d6qrtxccwn8gy5zuxvnbx5.streamlit.app/

## Переменные окружения (Yandex Cloud)

Для **протокола** и **черновика SpeechKit** используются одни и те же учётные данные, что и в `protocol.py`:

- `YANDEX_FOLDER_ID` (или `YC_FOLDER_ID`) — опционально для SpeechKit, но рекомендуется; передаётся в gRPC как `x-folder-id`.
- `YANDEX_CLOUD_API_KEY` (или `YC_API_KEY` / `YANDEX_API_KEY`) **или** `YANDEX_IAM_TOKEN` / `YC_IAM_TOKEN`.

Сервисному аккаунту в каталоге нужны роли на **Foundation Models** (протокол) и **`ai.speechkit-stt.user`** (потоковое STT). Расходы **SpeechKit** и **YandexGPT** считаются отдельно по тарифам облака.

## Возможности

- **Черновик SpeechKit:** блок WebRTC вверху страницы — включите запись, говорите; текст обновляется приблизительно в реальном времени. Это только подсказка; в протокол не попадает.
- Запись консультации **несколькими фрагментами** (`st.audio_input`: стоп микрофона → «Добавить фрагмент»), затем кнопка **«Транскрибировать и заполнить протокол»** — **Whisper** по склеенному WAV и **YandexGPT** по этому транскрипту.
- Длинное суммарное аудио при Whisper **режется на части** (по умолчанию до 30 секунд на кусок).
- Скачивание готового протокола: **protocol.docx** и **protocol.txt** (после заполнения также пробуется автозагрузка `protocol.txt` в браузере).

### WebRTC и продакшен

Для черновика используется **streamlit-webrtc** и публичный STUN (Google). За жёстким NAT или в закрытых сетях может понадобиться **свой TURN-сервер** и передача `rtc_configuration` с ICE-серверами — см. документацию [streamlit-webrtc](https://github.com/whitphx/streamlit-webrtc).

## Требования к системе

- Python 3.8+
- ~2GB disk space for Whisper models

## Как пользоваться?

1. Запишите фрагмент микрофоном и остановите запись.
2. Нажмите **«Добавить фрагмент в консультацию»** (при необходимости повторите для следующих частей).
3. Нажмите **«Транскрибировать и заполнить протокол»** (если последний фрагмент ещё не добавлен, он будет учтён автоматически).
4. Проверьте транскрипт и протокол, при необходимости отредактируйте.
5. При необходимости скачайте протокол в **.docx** или **.txt**.
